# Memory 시스템 가이드

전문가 에이전트의 지속적 학습을 위한 Memory 시스템 설계 가이드입니다.

## 디렉토리 구조

```
memory/
├── MEMORY.md               # 엔트리포인트 (200줄 이내)
├── task-log.md              # 작업 이력
├── lessons-learned.md       # 학습 기록
└── user-preferences.md      # 사용자 선호도
```

## 각 파일의 역할

### MEMORY.md (엔트리포인트)
- **200줄 이내** 유지 (초과 시 자동 로딩 범위를 벗어남)
- 최신 핵심 학습 사항을 상위에 배치
- 중요도 순으로 정렬, 오래된 내용은 하위 파일로 이동
- 포함 내용: 핵심 학습 요약, 자주 하는 실수와 해결법, 사용자 핵심 선호도, 다른 Memory 파일 참조 링크

### task-log.md (작업 이력)
- 수행한 작업의 시간순 기록
- 형식: `[날짜] 작업명 - 결과 요약 - 특이사항`
- subagent 호출 기록 형식: `[날짜] subagent:{name} - 전달 컨텍스트 요약 - 결과 요약 - 특이사항`
- 반복 패턴 발견 시 MEMORY.md에 요약

### lessons-learned.md (학습 기록)
- 실수와 조치 내역
- 발견한 패턴과 인사이트
- 형식: `[날짜] 상황 → 문제 → 해결 → 교훈`

### user-preferences.md (사용자 선호도)
- 선호하는 톤, 형식, 깊이
- 자주 요청하는 작업 패턴
- 싫어하는 것, 피해야 할 것

## Memory 관리 원칙

1. **최신 우선**: 가장 최근 학습이 MEMORY.md 상단에 위치
2. **중복 제거**: 같은 교훈이 반복되면 하나로 통합하고 빈도 표시
3. **주기적 정리**: 50개 이상 항목 누적 시 오래된 것을 요약/아카이브
4. **실행 가능한 형태**: "~했다"가 아닌 "~해야 한다" 형태로 기록
5. **크기 제한**: MEMORY.md 200줄 이내, 개별 파일 500줄 이내

## Memory vs Knowledge 구분

Memory와 Knowledge는 보완 관계이며, 목적과 관리 방식이 다릅니다.

| 구분 | Memory | Knowledge |
|------|--------|-----------|
| 목적 | 경험 기록 (무엇을 겪었는가) | 도메인 지식 축적 (무엇을 아는가) |
| 내용 | 작업 이력, 실수, 교훈, 선호도 | 개념, 프레임워크, 베스트 프랙티스 |
| 트리거 | 모든 작업 후 | 웹 서칭 + 카테고리 부합 + 새 내용 |
| 크기 제한 | MEMORY.md 200줄 | topic별 제한 없음 |
| 관리 방식 | 오래된 것 아카이브/탈락 | 오래된 것 재검증 |
| 기록 시점 | 작업 완료 후 | 대화 중 즉시 (웹 서칭 직후) |
