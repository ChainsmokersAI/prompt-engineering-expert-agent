# User Inputs 시스템 가이드

사용자가 제공한 원본 자료를 보관하고 참조하기 위한 User Inputs 시스템 설계 가이드입니다.

## 3체계 개념

전문가 에이전트는 3가지 정보 체계를 운영합니다:

- **Memory**: 에이전트의 경험 기록 (작업 이력, 학습, 선호도)
- **Knowledge**: 에이전트가 수집·정리한 도메인 지식 (프레임워크, 베스트 프랙티스)
- **User Inputs**: 사용자가 제공한 원본 자료 (에이전트 가공 없이 보관)

| 구분 | Memory | Knowledge | User Inputs |
|------|--------|-----------|-------------|
| 정체성 | 에이전트의 경험 기록 | 에이전트가 수집한 도메인 지식 | 사용자가 제공한 원본 자료 |
| 작성 주체 | 에이전트 | 에이전트 | 사용자 |
| 가공 여부 | 에이전트가 정리 | 에이전트가 정리 | 원본 그대로 보존 |
| 참조 트리거 | 세션 시작, skill 실행 전 | 모든 작업 시작 전 | 모든 작업 시작 전 (index.md) |
| 관리 방식 | 오래된 것 탈락 | 오래된 것 재검증 | 사용자 요청 시에만 삭제 |
| 크기 제한 | MEMORY.md 200줄 | 없음 | 없음 |
| 기록 시점 | 작업 완료 후 | 대화 중 즉시 (웹 서칭 직후) | 사용자 자료 제공 시 |

## 디렉토리 구조

```
user-inputs/
├── index.md                        # 분류 목록 및 자료 메타 정보 (엔트리포인트)
├── {category-1}/                   # 도메인별 분류 디렉토리
│   └── {filename}.md               # 개별 자료 파일
├── {category-2}/
│   └── {filename}.md
└── ...
```

### 분류 디렉토리 예시

분류 체계는 도메인에 따라 자유롭게 정의합니다:

- **스타트업 전문가**: `customer-research/`, `product-tech/`, `market-data/`
- **투자 전문가**: `portfolio-data/`, `investment-thesis/`, `financial-reports/`
- **커리어 전문가**: `resume/`, `job-descriptions/`, `interview-notes/`
- **AI 전문가**: `datasets/`, `model-configs/`, `experiment-logs/`

> 분류 체계는 전문가 생성 시 도메인 특성에 맞게 사전 정의하며, 필요 시 사용자와 협의하여 추가합니다.

## 저장 규칙

1. **원본 보존**: 사용자가 제공한 자료를 에이전트가 가공하지 않습니다 (헤더 추가, 내용 정리, 구조화 금지)
2. **삭제 금지**: 시간 경과에 따른 자동 탈락 없음. 사용자가 명시적으로 삭제를 요청한 경우에만 삭제합니다
3. **파일명 규칙**: 내용을 식별할 수 있는 간결한 이름 사용 (예: `customer-interview-2026-03.md`)
4. **크기 제한 없음**: 원본 자료의 크기를 제한하지 않습니다

## 에이전트 참조 기준

### 참조 시점
- **모든 작업 시작 전**, `user-inputs/index.md`를 확인합니다
- index.md에서 현재 작업과 관련된 자료가 있는지 판단합니다

### 조건부 참조
- 관련 자료가 있으면: 해당 파일을 Read하여 작업에 반영합니다
- 관련 자료가 없으면: 참조 없이 작업을 진행합니다
- 판단 기준: "이 자료가 현재 작업의 품질을 높이는가?"

### 참조 시 주의사항
- 원본 자료의 내용을 그대로 인용하거나 반영합니다 (자의적 재해석 금지)
- 자료 간 상충이 있으면 사용자에게 확인합니다
- 오래된 자료라도 삭제하지 않고, 필요 시 사용자에게 최신 여부를 확인합니다

## 자료 등록 워크플로우

사용자가 자료를 제공하면 다음 절차를 따릅니다:

1. **분류 판단**: 기존 분류 디렉토리 중 적합한 곳을 선택합니다. 없으면 새 분류를 생성합니다
2. **파일 저장**: 원본 그대로 해당 분류 디렉토리에 저장합니다
3. **index.md 업데이트**: 새 자료의 메타 정보를 추가합니다
4. **사용자 확인**: 저장 결과를 사용자에게 보고합니다

## index.md 작성 규칙

### 구조

```markdown
# User Inputs

사용자가 제공한 원본 자료입니다.

## 분류 체계

| 분류 | 설명 | 자료 수 |
|------|------|---------|
| [{category}]({category}/) | {한 줄 설명} | 0 |

## 자료 목록

| 파일 | 분류 | 등록일 | 설명 |
|------|------|--------|------|
| [{filename}]({category}/{filename}.md) | {category} | {YYYY-MM-DD} | {한 줄 요약} |
```

### 관리 원칙
- 분류 체계는 전문가 생성 시 도메인에 맞게 사전 정의합니다
- 자료 목록은 등록 시간순으로 최신이 상단에 위치합니다
- 자료 수는 실제 파일 수와 일치시킵니다

## 초기 세팅 절차

새 전문가를 생성할 때:

1. 도메인에 맞는 분류 체계를 2-3개 정의합니다
2. `user-inputs/index.md` 생성 (분류 목록 + 빈 자료 목록)
3. 각 분류 디렉토리 생성
4. CLAUDE.md에 User Inputs 참조 규칙 포함 (규칙 5)
